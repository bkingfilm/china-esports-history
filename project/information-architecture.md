# 信息架构与迁移设计

## 目标与边界

本次整理把“原始史料”“站点入口”“维护数据”分开管理。正文仍是作者原文，不因栏目、SEO 或风格统一而改写；分类、年代和实体关联仅作为检索层，可随研究深入而修订。

## 目录

```text

├─ assets/                  # 所有 locale 共用的图片与附件
├─ zh/
│  ├─ index.md              # 简体中文首页
│  ├─ chapters/             # 《中国电竞幕后史》48 篇 + 栏目索引
│  ├─ gamehistory/          # 《中国游戏幕后史》10 篇 + 栏目索引

└─ en/
   ├─ index.md              # 英文首页
   └─ one-more-way-…md      # 已发布译文

├─ content-map.csv          # 67 篇内容的权威映射与翻译状态
├─ information-architecture.md
├─ localization.md

```

`docs/` 内共有 75 个 Markdown 页面：67 篇历史内容、3 个中文栏目索引、2 个语言首页、1 个语言选择页、1 篇英文译文和 1 个许可页。英文 TEDx 译文是中文序章的 locale counterpart，不计入 67 篇中文原始内容。

## 稳定标识与路径

- `project/content-map.csv` 的 `content_id` 是跨语言稳定标识，不能因标题、文件名或栏目调整而改变。
- 中文原文路径采用 `docs/zh/<section>/<原文件名>.md`。第一阶段保留原文件名，避免一次变更同时改变目录与 slug。
- 译文放在 `docs/<locale>/<stable-slug>.md`，并通过映射表的 `<locale>_path` 与同一 `content_id` 关联。
- 新 locale 使用小写 BCP 47 标签；例如 `zh-Hant` 对应目录 `zh-hant/`。
- 共享资源只存一份，位于 `docs/assets/`。正文不得复制一套语言专属图片，除非图片本身包含需要本地化的文字。

## 浏览模型

站点导航保留作者的成书顺序；这是一条稳定、可审查的主线。辅助浏览由中文首页和三个栏目索引提供：

- 按年代：早期电竞、2003—2008 职业化、2009—2014 平台与直播、2015—2018 移动电竞与回顾；
- 按主题：人物、战队、赛事、媒体、平台、公司、社区与文化；
- 按实体：人物、战队、赛事和组织写入映射表 `entities` 字段，可供后续生成索引。

这些字段是导航 metadata，不改变正文中的历史表述。无法确定时使用“跨年代”或较宽的时期，不制造精确日期。

## 命名与链接规则

- Markdown 文件使用 UTF-8、一个一级标题和 `.md` 扩展名。
- 新英文 slug 使用小写 ASCII kebab-case；现有中文文件名在没有重定向机制前保持不变。
- 中文正文引用共享资源：`../../assets/...`；语言首页引用：`../assets/...`。
- 同 locale 内部链接使用相对 `.md` 路径；跨 locale 链接必须明确指向目标 locale，不能依赖目录猜测。
- 外部来源保留 `https://` 完整地址。移动文件后必须运行链接检查。

## 迁移策略

67 篇内容全部使用 Git 感知的移动保留历史。正文变化仅限迁移后必须调整的相对图片或 counterpart 链接。原始路径、标题、分类、时期、实体、新路径、迁移状态和翻译状态见 `content-map.csv`。

若未来需要改成全 ASCII 稳定 URL，应先设计重定向表，再单独提交；不能把路径清理与内容修改混入同一 PR。

## 路线图

1. **当前阶段**：完成目录分层、67 篇映射、75 页构建、中文/英文入口与严格检查。
2. **索引阶段**：基于 `content-map.csv` 自动生成时间线、主题和实体详情页，避免手工维护重复列表。
3. **翻译阶段**：按读者请求选择章节，执行翻译—史实核对—语言校对流程。
4. **URL 阶段**：如上游认可，再引入稳定 ASCII 路由与旧路径重定向。
5. **史料阶段**：为来源、勘误和争议信息建立独立审查记录，不直接覆盖作者原文。
